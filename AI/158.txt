You're touching on a very profound conceptual point here, which is the difference between an infinite process or limit and the *definition* of a mathematical object within a given set.

Let's clarify why a natural number always has a finite number of digits, even when we talk about the set of natural numbers going "to infinity."

1.  **Definition of a Natural Number:**
    In standard set theory and arithmetic, a natural number is defined as an element of the set $\mathbb{N} = \{1, 2, 3, 4, \dots\}$ (or $\{0, 1, 2, 3, \dots\}$). Each individual number in this set is a *finite* quantity.
    * `1` is a natural number, it has 1 digit.
    * `100` is a natural number, it has 3 digits.
    * `1,000,000,000,000` is a natural number, it has 13 digits.
    * `googol` ($10^{100}$) is a natural number, it has 101 digits.
    * `googolplex` ($10^{\text{googol}}$) is a natural number, it has googol + 1 digits.

    No matter how large a natural number you pick, you can always write it down with a finite string of digits. The number of digits might be astronomically large, but it's always a specific, finite count.

2.  **The "Infinity" of Natural Numbers:**
    The "infinity" of the set of natural numbers ($\mathbb{N}$) refers to the fact that the set is **unbounded** and **endless**. For any natural number $n$ you choose, there is always a successor $n+1$. This means you can keep generating larger and larger natural numbers forever.
    * This *process* of generating numbers is infinite.
    * The *set itself* is infinite (countably infinite, as we discussed).
    * But **each individual element** within that infinite set is a finite number.

    Think of it like this: Imagine a very long line of cars. The line of cars goes on forever. But each individual car in that line is a finite object; it doesn't have an infinite length. The "infinity" applies to the *number of cars*, not the size of any *single car*.

3.  **Why a Natural Number Cannot Have Infinite Digits:**
    If a number were to have an infinite number of digits, it would imply something fundamentally different from a natural number.
    * Consider $1.000...$ (repeating 0s). This is just $1$. It's a rational number, and it's a natural number.
    * Consider $0.333...$ (repeating 3s). This is $1/3$. It's a rational number, but not a natural number. It has infinite digits after the decimal point.
    * Consider $\pi = 3.14159265...$. This is an irrational number with infinite, non-repeating digits.

    If we allowed natural numbers to have infinite digits, then the distinction between natural numbers and real numbers would blur, and the entire framework of number theory and analysis would collapse. The number 314159... (with infinite digits) *is* essentially the real number $\pi$ (or $\pi \times 10^k$ for some $k$). It's not a natural number.

4.  **"Even at infinity we can get next number by just adding one more":**
    This phrase still refers to the property of the *set* of natural numbers being unbounded. It means that no matter how far "out" in the sequence of natural numbers you go, you can always find a next one. It doesn't mean that a *single* natural number at the "end" of infinity has infinite digits. There is no "end" or "last" natural number where this property would manifest.

**In summary:**

* **Individual natural numbers are finite.** They can be arbitrarily large, but they always have a finite, countable number of digits.
* **The set of natural numbers is infinite** (countably infinite) because there's always a successor, meaning you can list them without end.
* The concept of a number with an "infinite number of digits" is typically reserved for real numbers (especially irrational ones or rational numbers with non-terminating decimal expansions), which belong to a larger, uncountably infinite set.

The distinction between a set being infinite and its individual elements being finite is crucial.
